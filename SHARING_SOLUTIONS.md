# 도서관 공유 기능 해결 방안

## 문제점
현재 localStorage 기반 공유는 브라우저별로 격리되어 다른 사용자와 공유가 불가능합니다.

## 해결 방안

### 방법 1: JSON 파일 다운로드/업로드 (가장 간단) ⭐ 추천
**장점:**
- 즉시 구현 가능
- 서버 불필요
- 무료
- 데이터 크기 제한 없음

**단점:**
- 수동 다운로드/업로드 필요
- 실시간 동기화 불가

**구현 방법:**
1. "도서관 내보내기" 버튼으로 JSON 파일 다운로드
2. 다른 사용자가 "도서관 가져오기" 버튼으로 JSON 파일 업로드
3. 공개 설정에 따라 정보 필터링

### 방법 2: GitHub Gist API (추천) ⭐⭐
**장점:**
- 무료
- 자동 동기화 가능
- URL로 바로 공유 가능
- GitHub 계정만 있으면 됨

**단점:**
- GitHub API 토큰 필요
- 공개 Gist는 누구나 볼 수 있음

**구현 방법:**
1. 사용자가 GitHub Personal Access Token 입력
2. 도서관 데이터를 Gist로 저장
3. Gist ID를 URL에 포함하여 공유
4. 다른 사용자가 Gist ID로 데이터 로드

### 방법 3: Firebase (장기적 해결책) ⭐⭐⭐
**장점:**
- 실시간 동기화
- 사용자 인증
- 확장성 좋음
- 무료 티어 제공

**단점:**
- 초기 설정 필요
- Firebase 프로젝트 생성 필요

**구현 방법:**
1. Firebase 프로젝트 생성
2. Firestore 데이터베이스 설정
3. 인증 시스템 구축
4. 실시간 데이터 동기화

### 방법 4: URL 파라미터 인코딩 (제한적)
**장점:**
- 즉시 구현 가능
- 서버 불필요

**단점:**
- URL 길이 제한 (약 2000자)
- 작은 데이터만 가능
- 브라우저 히스토리 노출

## 추천 구현 순서

1. **즉시**: JSON 파일 다운로드/업로드 기능 추가
2. **단기**: GitHub Gist API 통합
3. **장기**: Firebase로 전환
